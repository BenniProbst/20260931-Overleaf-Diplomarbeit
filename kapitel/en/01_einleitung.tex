\section{Motivation}\label{sec:motivation}
Search trees and index structures underpin modern data-intensive systems --- from in-memory databases through
file systems to language processing. At the same time, today's processors are dominated by the memory hierarchy:
not the number of operations but their cache behaviour determines run time. From this tension arises the guiding
question of this thesis.
\bildPH{B1: Cache wall}{Latency pyramid registers$\to$L1$\to$L2$\to$L3$\to$RAM; the widening gap motivates cache-aware search structures.}

\section{Problem Statement}\label{sec:problem}
In today's search structures cache-awareness is either hard-wired or ignored entirely; above all, however, one
cannot \emph{separate} which individual design decision benefits which load pattern --- the \emph{separability
problem}, the central thread of this thesis.
\bildPH{B2: Separability problem}{Monolithic algorithm (black box) vs. algorithm decomposed into axes: which component contributes to performance?}

\section{Structure of the Thesis}\label{sec:structure}
The structure follows the path from the general to the specific: Chapter~\ref{ch:foundations} lays --- conceptually
