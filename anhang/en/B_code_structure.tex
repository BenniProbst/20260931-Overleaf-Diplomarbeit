\chapter{Code Structure}\label{app:code}

This appendix documents the real source-code structure of the three repositories from
Section~\ref{sec:repos} in a verifiable way at directory level. The code state was surveyed on
11~July~2026 via directory listings (two to three levels deep); the authoritative commits are
\texttt{c022ce05} (\texttt{comdare-cache-engine}), \texttt{faa4e76} (\texttt{comdare-prt-art}),
and \texttt{ba3a9ae} (\texttt{probst-\allowbreak{}Diplomarbeit-\allowbreak{}cache-\allowbreak{}engine},
super-repository). Build
artefacts (\texttt{build*/}), IDE directories, and the vendored third-party code under
\texttt{ext/} are hidden or collapsed into a single collective entry; all other paths listed
here exist in the surveyed state. Paths are given relative to the respective repository root.

\section*{comdare-cache-engine --- building-block library and builder}

The repository follows the Pitchfork layout: \texttt{apps/} contains the executables,
\texttt{libs/} the libraries, and \texttt{libs/common} the cross-cutting code. The vendored
third-party code of the A/P series is collected under \texttt{ext/} and is not expanded here.

\texttt{prt\_art/\allowbreak{}include/\allowbreak{}prt\_art/}; each building block resides in

evaluation pipeline, and the manuscript; the two code repositories are embedded as submodules
under \texttt{Code/\allowbreak{}external/}.
